Kibana - Data Visualization & Analysis Platform

Kibana adalah bagian dari Elastic Stack yang menyediakan interface untuk data visualization, search, dan analysis. Sangat powerful untuk log analysis, monitoring, dan business intelligence.

Apa itu Kibana?

Kibana adalah web interface yang memungkinkan Anda untuk memvisualisasikan data dari Elasticsearch. Dengan Kibana, Anda dapat membuat dashboard interaktif, melakukan pencarian data, dan menganalisis data dalam waktu nyata.

Fitur Utama Kibana

  • Discover: Eksplorasi dan pencarian data
  • Visualize: Membuat chart dan grafik
  • Dashboard: Kumpulan visualisasi
  • Maps: Visualisasi geografis
  • Machine Learning: Anomaly detection
  • Dev Tools: Console untuk Elasticsearch

Kapan Menggunakan Kibana

  • Log analysis dan monitoring
  • Business intelligence
  • Security analysis (SIEM)
  • Application performance monitoring
  • Real-time data visualization
  • Geographic data analysis

Setup & Installation

Instalasi Kibana

Dengan Docker (Rekomendasi)
# Pull Kibana image
docker pull kibana:8.11.0

# Run Kibana dengan Elasticsearch
docker run -d \
  --name kibana \
  -p 5601:5601 \
  -e ELASTICSEARCH_HOSTS=http://elasticsearch:9200 \
  kibana:8.11.0

# Atau dengan docker-compose
version: '3.8'
services:
  elasticsearch:
    image: elasticsearch:8.11.0
    environment:
      - discovery.type=single-node
      - xpack.security.enabled=false
    ports:
      - "9200:9200"
  
  kibana:
    image: kibana:8.11.0
    environment:
      - ELASTICSEARCH_HOSTS=http://elasticsearch:9200
    ports:
      - "5601:5601"
    depends_on:
      - elasticsearch
Instalasi Manual
# Download Kibana
wget https://artifacts.elastic.co/downloads/kibana/kibana-8.11.0-linux-x86_64.tar.gz

# Extract
tar -xzf kibana-8.11.0-linux-x86_64.tar.gz
cd kibana-8.11.0-linux-x86_64

# Start Kibana
./bin/kibana

# Atau sebagai service
sudo systemctl start kibana
sudo systemctl enable kibana
Konfigurasi Dasar
# File: config/kibana.yml
server.port: 5601
server.host: "0.0.0.0"
elasticsearch.hosts: ["http://localhost:9200"]

# Security (opsional)
elasticsearch.username: "kibana_system"
elasticsearch.password: "password"

# Logging
logging.level: "info"
logging.appenders.file.type: file
logging.appenders.file.fileName: /var/log/kibana/kibana.log

Dev Tools - Elasticsearch Console

Query Console

Search Queries
# Basic search
GET /web_logs/_search
{
  "query": {
    "match_all": {}
  }
}

# Match query
GET /web_logs/_search
{
  "query": {
    "match": {
      "message": "error"
    }
  }
}

# Range query
GET /web_logs/_search
{
  "query": {
    "range": {
      "response_time": {
        "gte": 1000
      }
    }
  }
}
Aggregations
# Terms aggregation
GET /web_logs/_search
{
  "size": 0,
  "aggs": {
    "status_codes": {

        "field": "status"

  }
}

  "size": 0,

  }
}

Monitoring & Alerts

Watcher Alerts
# Create watcher
PUT _watcher/watch/error_alert
{
  "trigger": {
    "schedule": {
      "interval": "1m"
    }
  },
  "input": {
    "search": {
      "request": {
        "search_type": "query_then_fetch",
        "indices": ["logs-*"],
        "body": {
          "query": {
            "bool": {
              "must": [
                { "match": { "level": "ERROR" } },
                { "range": { "@timestamp": { "gte": "now-1m" } } }
              ]
            }
          }
        }
      }
    }
  },
  "condition": {
    "compare": {
      "ctx.payload.hits.total": {
        "gt": 10
      }
    }
  },
  "actions": {
    "send_email": {
      "email": {
        "to": ["admin@example.com"],
        "subject": "High Error Rate Alert",
        "body": "Error count: {{ctx.payload.hits.total}}"
      }
    }
  }
}
